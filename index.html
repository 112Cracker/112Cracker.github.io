Posted on 2018-12-31 | In Software Design

Design Patterns - Overview

在软件工程领域，设计模式通常是针对特定的设计情景提出的一种普适的，可重复使用的解决方案。程序开发者在实践中不断地试错，针对一类具有共性的设计问题提出所谓“最佳”的设计方法，这些方法便是设计模式。在OOP程序设计中，设计模式通常通过定义不同的类和实例的关系以及相互作用来达到更好的设计出扩展性强，灵活性高，可维护性优良的应用或系统。

在Design Patterns - Elements of Reusable Object-Oriented Software这部经典著作中，作者提出了OOP设计模式的两个基本原则：

  • Program to an interface not an implementation.
  • Favor object composition over inheritence.

设计模式为程序开发者提供了一种通用的沟通语言，这表现在这些设计模式都有着标准化的术语名称，并可以用来解决一类共性的问题。这样，程序开发者可以很直接明了的通过某个设计模式的名称，如Singleton Pattern了解到他人程序设计的思路与方法。

在Design Patterns - Elements of Reusable Object-Oriented Software中，作者总结了23种OOP的设计模式，尽管实际的设计模式数量远远不止于此，这23种设计模式是每个程序开发者都应该了解，并在实践中灵活应用的。这23种设计模式可以总结为三个大类。

  • Creational Patterns: 这类设计模式提供了一种这创建实例的过程中隐藏创建逻辑的方法，即不直接通过实例类的构造器来创建实例。
  • Structural Patterns：这类设计模式重点在于类与实例之间的组合。
  • Behavioral Patterns：这类设计模式重点在于实例之间的相互作用。

Factory Pattern (Creational Pattern)

Factory Pattern是Java中应用最广泛的设计模式之一，它属于Creational Pattern的一种。利用Factory Pattern，我们可以向Client隐藏构建实例的逻辑，Client通过Facotry Method来构建实例，不需要通过具体的实例类的构造器来完成。这些Factory Method既可以通过一个接口来定义并由一个类实现；也可以通过一个抽象的类来实现并有一个具体类来重写。

Abstract Factory Pattern (Creational Pattern)

Abstract Factory Pattern提供了一种封装一组Factory的方法，这组Factory往往被设计成可以“生产”同一主题的“产品”，然而这类“产品”之间在实现细节和具体表现形式上却各有不同

举个例子，假设我们有一个AbstractRobotFactory（一个抽象类），这个类中，包括两个方法：createRobotHead()和 createRobotArm()。那么，如果我们想得到一个R2-D2和一个C-3PO就需要不同的Concrete Factory类来继承并实现这个AbsractRobotFactory，R2D2Factory()和C3POFactory 可以通过对creatRobotHead() 和 createRobotArm() 这两个方法（Abstract Method）的不同实现来满足自己的设计要求。同时，我们还可以构建两个接口，Head和Arm 来隐藏具体类，比如RoundHead，SquareHea或者ShortArm，LongArm。

通过这种设计模式，我们可以向Client隐藏具体类的实现逻辑，通过继承（inheritence）和实例的组合（composition），我们使大量实例的实现和使用彼此分离，使代码具有了更好的扩展性和灵活性。

Spring Boot 介绍

Sprint Boot是Spring开发的众多项目之一。通过Spring Boot，开发者可以使用Java语言便捷高效地进行服务器端的Web开发。尽管Spring Boot不是唯一的基于Java的框架，开发者利用Spring Boot可以简洁高效地开发独立的，可投入生产的，基于Spring生态的应用，正如官方文档中所说，”Applications that you can ‘just run’”。Spring Boot在适用于编写小型应用解决实际问题的同时，其真正的强项在于使用Spring Cloud构建大型的，分布式的应用程序，在这种情境下，通常有多个应用程序进行协同工作。

Spring Boot 2.0 的新特性：

  • 编程语言：Java 8+, Kotlin
  • 底层框架：Spring Framework 5.0.x
  • 全新特性： Web Flux

为什么选择Web Flux?

  • 函数编程：Java 8 Lamdba
  • 响应式编程：Reative Streams
